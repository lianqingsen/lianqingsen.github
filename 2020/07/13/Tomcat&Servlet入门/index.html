Tomcat&Servlet入门

Web相关概念

  1. 软件结构

    1. c/s ：客户端/服务端 Client/Service
    2. b/s：浏览器/服务器 borwser/service

  2. 资源分类

    1. 静态资源:所有用户访问，得到的结果都是一样的，称为静态资源。静态资源可以在浏览器直接解析

      如：html，css，JavaScript

    2. 动态资源：每个用户访问同资源后，得到的结果不一样的，称为动态资源。动态资源需要转化为静态资源，再返回给浏览器。

      如:servlet/jsp,php,asp….

  3. 网络通信三要素

    1. ip
    2. 端口（port）: 一台计算机有0-65536个端口
    3. 传输协议：规定传输协议
      1. 基础协议
        1. tcp：安全协议，三次握手，四次挥手。速度较慢
        2. udp：不安全协议：直接向服务端连接。速度快。

web服务器软件

  • 服务器：安装了服务器软件计算器

  • 服务器软件：接受用户请求，处理请求，走出响应

    web服务器软件：是让用户通过浏览器来访问资源，做出响应

  • 常见Java相关的web服务器软件：

    • webLogic:（oracle公司），大型JavaEE服务器，支持所有JavaEE规范，收费的

    • webSphere：IBM公司，大型JavaEE服务器，支持所有JavaEE规范，收费的

    • JBOSS：JBOSS公司，大型JavaEE服务器，支持所有JavaEE规范，收费的

    • Tomcat: Apche基金组织，中小型的JavaEE服务器，仅仅支持小量的JavaEE规范servlet/jsp，开源 、免费的

  • JavaEE：java语言在企业开发中使用技术规范综合，一共规定13项大的规范。

  • Tomcat：web服务器软件

    1. 下载、安装、卸载 (忽略)

    2. 启动：

      • 在bin目录里面找到startup.bat 启动，bat是windows的执行文件，而startup.sh是linux的执行文件

      • 默认8080端口，浏览器输入：http://localhost:8080

      • 可能遇到的问题

        1. 黑窗口一闪而过

          原因：没有正确配置JDK

        2. 启动报错 原因：端口号被占用

          解决方案：

          1. 暴力：找到占用端口号，并且找到对应的进程，杀死该进程

            cnd 查看端口： netstat -ano

          2. 温柔：修改自身的端口

          一般会将tomcat的默认端口号修改为80。80端口号是http协议的默认端口号

          • 好处在http://localhost，后面不需要加端口号。

    3. 关闭：

      1. 正常关闭
        • 第一种：bin/shutdown.bat
        • 第二种：ctrl+c
      2. 强制关闭
        • 直接点击窗口的x

    4. 配置：

      部署项目方式：

      1. 直接将项目放到webapps目录下即可

        简化部署：将项目打包成一个war包，再将war包放置到webapps目录下

        • war包会自动解压缩

  1. 静态项目和动态项目

    java动态项目的目录结构

    项目的根目录 web-inf目录：

    • web.xml:web项目的核心配置文件
    • classes目录：放置字节码文件的目录
    • lib目录：放置依赖的jar包

servlet： service applet

  • 概念：运行在服务器端的小程序

    servlet就是一个接口，定义了java类被浏览器访问到tomcat识别的规则

    将来我们自定义一个类，实现servlet接口，覆写方法。

  • 快速入门

    1. 创建JavaEE项目

    2. 定义一个类，实现Servlet接口

    3. 实现接口的抽象方法

    4. 配置Servlet

  • Servlet_生命周期：首先加载servlet的class，(newInstance)实例化servlet，然后初始化servlet调用init()的方法，接着调用服务的service的方法处理doGet和doPost方法，最后是我的还有容器关闭时候调用destroy 销毁方法。

  • ServLet中的生命周期

    1. 被创建，执行init方法进行初始化，只执行一次

      • Servlet什么时候被创建？

        默认情况下，第一次被访问时，Servlet被创建

        可以配置执行Servlet的创建时机。

        • 在标签下配置
        1. 第一次发访问是，创建 的值为负数
        2. 在服务启动时，创建 的值为0或正数

      • Servlet执行init方法，只执行一次，说明一个Servlet内存只存在一个对象，Servlet是单例的

        • 多个用户访问时，可以存在线程安全
        • 解决：尽量不要在Servlet中定义成员变量，即使定义了成员变量，也不要对修改值。

    2. 提供服务：执行servlet方法来处理客户端的请求，执行多次

    3. 被销毁：执行destory方法终止，执行一次

      • Servlet被销毁时执行，服务器关闭时，Servlet被销毁时
      • 只有正常关闭服务器时，才会执行destory()方法

    4. 最后，Servlet 是由 JVM 的垃圾回收器进行垃圾回收的。

  • Servlet3.0:

    • 好处：
    • 支持注解配置，可以不需要web.xml
    • 步骤：
      1. 创建JavaEE，选择Servlet的版本3.0以上，可以不创建web.xml
      2. 定义一个类，实现Servlet接口
      3. 复写方法
      4. 在类上使用@WebServlet注解，进行配置
        • @WebServlet(“资源路劲”)

    • GenericServlet ：将Servlet接口中其他的方法做了默认空实现，只将Service()方法作为抽象

    • urlPattern：Servlet访问路劲
      1. urlPattern是一个数组，可以定义多个Servlet访问路劲：@WebServlet({“/d3”,”/d4”,”/d5”})
      2. 路劲定义规则：

        2. /xxx/xxx：多层路劲，目录结构
        3. .do : @WebServlet(“.do”)

HTTP:

概念：Hyper Text Transfer Protocol 超文本传输协议

  • 传输协议：定义了，客户端和服务器通信时，发送数据的格式

  • 特点

    1. 基于TCP/IP的高级协议
    2. 默认端口号:80
    3. 基于请求/响应模型的一次请求对于一次响应。
    4. 无状态的：每次请求之间相互独立，不能交互数据

  • 历史版本（了解）：

    • 1.0 ：每一次请求响应都会建立新的连接 —– 单例设计模式
    • 1.1：复用连接

  • 请求消息数据格式

    1. 请求行

      请求方式 请求url 请求协议/版本

      GET filename: /firstweb/longin.html HTTP/1.1

      http协议有7中请求方式，常用有两种

      • get:

        1. 请求参数在请求行中，在url后。
        2. 请求的url的长度有限制的
        3. 不太安全

      • post：

        1. 请求参数在请求体中。

        2. 请求的url的长度没有限制的

        3. 相对安全

    2. 请求头

      请求头名 : 请求头值

      • 常见的请求头：
        1. user-Agent：浏览器告诉服务器，我访问你使用的浏览器版本信息
          • 可以在服务器获取该头的信息，解决浏览器的兼容问题。
        2. Referer：

    3. 请求空行

      空行，就用分割post请求的请求头，和请求体

    4. 请求体

      • 封装Post请求消息请求参数的

  • 字符串格式：
